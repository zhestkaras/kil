Вот две диаграммы в формате, аналогичном вашему примеру:  

---

### **Диаграмма АС ИС (Как есть):**
**Процесс выполнения самостоятельной работы**

**Задача 1: Выдача задания**  
- Преподаватель формулирует задание (1 день)  
- Преподаватель публикует задание в системе (0.5 дня)  
- Куратор уведомляет студентов о новом задании (0.5 дня)  

**Задача 2: Выполнение задания студентом**  
- Студент изучает материалы (2 дня)  
- Студент выполняет задание (3 дня)  
- Студент загружает работу в систему (0.5 дня)  

**Задача 3: Проверка и оценка**  
- Преподаватель проверяет работу (2 дня)  
- Преподаватель выставляет оценку (0.5 дня)  
- Куратор фиксирует результаты в отчёте (1 день)  

**Задача 4: Обратная связь**  
- Преподаватель отправляет комментарии студенту (0.5 дня)  
- Студент исправляет работу (при необходимости) (1 день)  
- Куратор контролирует исправление (0.5 дня)  

---

### **Диаграмма ТУ БИ (Как будет):**
**Оптимизированный процесс выполнения самостоятельной работы**

**Задача 1: Автоматизированная выдача задания**  
- Преподаватель загружает задание в систему (0.5 дня)  
- Система автоматически уведомляет студентов и куратора (0.1 дня)  
- Куратор проверяет доступность материалов (0.5 дня)  

**Задача 2: Контроль выполнения в реальном времени**  
- Студент работает в системе с таймером и прогрессом (3 дня)  
- Система автоматически напоминает о дедлайне (0.1 дня)  
- Куратор видит статистику выполнения (0.2 дня)  

**Задача 3: Автопроверка и оценка**  
- Система проверяет технические критерии (0.2 дня)  
- Преподаватель оценивает содержательную часть (1 день)  
- Оценка автоматически вносится в ведомость (0.1 дня)  

**Задача 4: Интегрированная обратная связь**  
- Преподаватель оставляет комментарии в системе (0.5 дня)  
- Студент вносит правки в тот же документ (1 день)  
- Куратор получает уведомление о завершении работы (0.1 дня)  

---

Обе диаграммы связывают трёх участников (куратор, преподаватель, студент) через чёткие задачи, исключая лишние бытовые действия.
